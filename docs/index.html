THE
SILENT
FAILURE

Certificates expire at 3 AM. Services go dark before anyone notices. SSH servers advertise decade-old key exchange algorithms. OIDC providers accept unsigned tokens. None of this triggers an alert.

Without continuous multi-protocol scanning, your certificate inventory is a snapshot from the last time someone checked manually — which was probably months ago.

▸ PrimeKey / EJBCA Integration

PULL FROM
THE CA

Network scanning only finds certificates that are reachable on a known port. EJBCA sync pulls every issued certificate directly from the CA — including internal micro-services, staging certs, and certs not yet deployed.

  • REST API v1 — POST /v1/search/certificates
  • mTLS (client cert + key) or API-key auth
  • Paginated — handles thousands of certs
  • CA DN filter: sync only selected CAs
  • Configurable interval (hours) or manual
  • EJBCA username → cert-guardian owner auto-map
  • Expiry alerts for EJBCA-only certs
  • Sync status in Settings: found / new / updated
